# 트랜지션

**트랜지션은 값이 한 상태에서 다른 상태로 애니메이션되는 방식을 정의합니다.**

`transition`은 두 값 사이의 애니메이션이 어떤 방식으로 동작할지를 정의합니다.

```jsx
<motion.div animate={{ x: 100 }} transition={{ delay: 1 }} />
```

<IFrameEmbed sandboxId="framer-motion-transition-prop-j9m7d2" />

트랜지션은 `Tween`, `Spring`, 또는 `Inertia` 중 어떤 유형의 애니메이션을 사용할지 정의하는 속성들도 받을 수 있습니다.

```jsx
<motion.div
  animate={{ x: 100 }}
  transition={{ type: "spring", stiffness: 100 }}
/>
```

### 값별 트랜지션

각각의 특정 값에 대해 서로 다른 트랜지션 설정을 정의할 수 있습니다.

```jsx
transition={{
  ease: "linear",
  duration: 2,
  x: { duration: 1 }
}}
```

## 오케스트레이션

#### delay: `number`

애니메이션을 지정된 시간(초)만큼 지연시킵니다. 기본값은 0입니다.

```jsx
const transition = {
  delay: 0.2,
};
```

delay를 음수 값으로 설정하면, 애니메이션이 해당 시간만큼 진행된 상태에서 시작됩니다. 예를 들어, 1초 지점에서 시작하려면 delay를 -1로 설정하면 됩니다.

#### delayChildren: `number`

variant를 사용할 때, 자식 애니메이션이 이 시간(초) 이후에 시작됩니다. 트랜지션 속성은 Frame과 variant 모두에 직접 추가할 수 있습니다. variant에 추가하는 것이 일반적으로 더 유연한데, 이는 시각적 상태별로 지연을 커스터마이즈할 수 있기 때문입니다.

```jsx
const container = {
  hidden: { opacity: 0 },
  show: {
    opacity: 1,
    transition: {
      delayChildren: 0.5,
    },
  },
};

const item = {
  hidden: { opacity: 0 },
  show: { opacity: 1 },
};

return (
  <motion.ul variants={container} initial="hidden" animate="show">
    <motion.li variants={item} />
    <motion.li variants={item} />
  </motion.ul>
);
```

#### staggerChildren: `number`

variant를 사용할 때, 자식 컴포넌트들의 애니메이션을 이 시간(초)만큼 순차적으로 지연시킬 수 있습니다.

예를 들어, staggerChildren이 0.01이면, 첫 번째 자식은 0초, 두 번째는 0.01초, 세 번째는 0.02초 등의 지연이 적용됩니다.

계산된 순차 지연은 delayChildren에 추가됩니다.

#### staggerDirection: `number`

자식들을 순차적으로 지연시키는 방향을 설정합니다.

1은 처음부터 마지막까지, -1은 마지막부터 처음까지 순차적으로 적용됩니다.

#### when: `false | "beforeChildren" | "afterChildren" | string`

트랜지션과 그 자식들 간의 관계를 설명합니다. 기본값은 false입니다.

variant를 사용할 때, 트랜지션은 자식들과의 관계에 따라 다음과 같이 예약될 수 있습니다:

- "beforeChildren": 자식 트랜지션을 시작하기 전에 이 트랜지션을 완료
- "afterChildren": 자식 트랜지션이 완료된 후 이 트랜지션을 시작

## 반복

#### repeat: `number`

트랜지션을 반복할 횟수입니다. 무한 반복을 위해서는 Infinity로 설정합니다.

repeatType를 설정하지 않으면 애니메이션이 반복됩니다.

#### repeatType: `"loop" | "reverse" | "mirror"`

애니메이션 반복 방식을 지정합니다:

- loop: 애니메이션을 처음부터 반복
- reverse: 정방향과 역방향 재생을 번갈아가며 실행
- mirror: from과 to 값을 번갈아가며 전환

#### repeatDelay: `number`

애니메이션을 반복할 때, 각 반복 사이에 대기할 시간(초)을 설정합니다.

## Tween

#### type: `"tween"`

지속 시간 기반 애니메이션을 사용하려면 type을 "tween"으로 설정합니다. 오케스트레이션이 아닌 트랜지션 값이 type 속성 없이 설정되면, 이것이 기본 애니메이션으로 사용됩니다.

#### duration: `number`

tween 애니메이션의 지속 시간입니다. 기본값은 0.3초이며, 키프레임 시리즈를 애니메이션할 때는 0.8초입니다.

#### ease: `Easing | Easing[]`

사용할 이징 함수입니다. 다음 중 하나로 설정할 수 있습니다:

- 기존 이징 함수의 이름
- cubic bezier 곡선을 정의하는 4개의 숫자 배열
- 0-1 사이의 값을 받아서 0-1 사이의 값을 반환하는 이징 함수

키프레임 애니메이션에서 여러 값의 배열로 애니메이션 값이 설정된 경우, ease를 이징 함수의 배열로 설정하여 각 값 사이에 서로 다른 이징을 설정할 수 있습니다.

Framer의 내장 이징 함수들:

- "linear"
- "easeIn", "easeOut", "easeInOut"
- "circIn", "circOut", "circInOut"
- "backIn", "backOut", "backInOut"
- "anticipate"

## Spring

현실적인 모션을 위한 스프링 물리학을 시뮬레이션하는 애니메이션입니다.

x, y, scale, rotate와 같은 물리적 값에 대한 기본 애니메이션입니다.

#### type: `"spring"`

스프링 물리학을 사용한 자연스러운 움직임을 위해 type을 "spring"으로 설정합니다. 기본값으로 설정되어 있습니다.

#### damping: `number`

반대 힘의 강도입니다. 0으로 설정하면 스프링이 무기한으로 진동합니다. 기본값은 10입니다.

#### mass: `number`

움직이는 물체의 질량입니다. 높은 값일수록 더 느린 움직임이 됩니다. 기본값은 1입니다.

#### stiffness: `number`

스프링의 강성입니다. 높은 값일수록 더 갑작스러운 움직임이 생성됩니다. 기본값은 100입니다.

## Inertia

초기 속도를 기반으로 값을 감속시키는 애니메이션으로, 주로 관성 스크롤을 구현하는 데 사용됩니다.

선택적으로 최소/최대 경계를 정의할 수 있으며, 관성은 스프링 애니메이션으로 이 경계에 스냅됩니다.

#### type: `"inertia"`

관성 애니메이션을 사용하려면 type을 "inertia"로 설정합니다. 모멘텀 스크롤링과 같은 자연스러운 감속에 사용될 수 있습니다.

#### power: `number`

높은 power 값은 더 먼 목표 지점을 의미합니다. 기본값은 0.8입니다.

#### timeConstant: `number`

시간 상수를 조정하면 감속의 지속 시간이 변경되어 느낌이 달라집니다. 기본값은 700입니다.

## 기타

#### TargetAndTransition

애니메이션할 대상 값을 지정하는 객체입니다. 각 값은 단일 값 또는 값의 배열로 설정할 수 있습니다.

다음 속성들도 포함할 수 있습니다:

- transition: 모든 값 또는 개별 값에 대한 트랜지션을 지정
- transitionEnd: 애니메이션이 끝날 때 설정할 값을 지정
